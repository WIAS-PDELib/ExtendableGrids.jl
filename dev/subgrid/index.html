Subgrid

Subgrids of an ExtendableGrid are again of the same type ExtendableGrid and unse the typed Dict mechanism to store linkage to the parent grid.

grid=simplexgrid([1,2,3], [4,5,6])
sub=subgrid(grid,[2],boundary=true, transform=(a,b) -> (a[1]=10*b[2]))
println(keys(sub))
println(sub[Coordinates])
Type{<:AbstractGridComponent}[CellNodes, BFaceNodes, BFaceGeometries, NumBFaceRegions, ExtendableGrids.NodeInParent, BFaceRegions, ExtendableGrids.ParentGrid, CoordinateSystem, Coordinates, CellRegions, CellGeometries]
[40 50 60]

Given a vector on the parent grid, one can create a view of this vecotor on the subgrid:

grid=simplexgrid([1,2,3], [4,5,6])
sub=subgrid(grid,[2],boundary=true, transform=(a,b) -> (a[1]=10*b[2]))
v=[i for i=1:num_nodes(grid)]
subv=view(v,sub)
println(subv)
[3, 6, 9]

ExtendableGrids.subgrid — Method
subgrid(
    parent,
    subregions::AbstractArray;
    transform,
    boundary,
    project
) -> ExtendableGrid

Create subgrid from list of regions.

  • parent: parent grid
  • subregions: Array of subregions
  • transform (kw parameter): transformation function between grid and subgrid coordinates acting on one point. Default: copytransform
  • boundary: if true, create codimension 1 subgrid from boundary region.
  • project: project coordinates onto subgrid dimension

A subgrid is of type ExtendableGrid and stores two additional components: ParentGrid and NodeInParent
